SCOPE AND NON-VACUITY
---------------------
Each scenario run drives a 3-node cluster for 8 seconds of virtual time with
6 simulated clients, 4 contended keys, and Get/Put/Delete/CAS operations.
Every batch completed more than 361,000 client operations. The full campaign
averaged approximately 2,081.6 completed operations per scenario run.

TestScenarioSmoke and TestScenarioRegression reject zero-operation runs.
TestScenarioSoak records aggregate batch operations rather than a per-run
minimum, so this artifact claims non-vacuity for every batch and for the full
campaign, not an unrecorded minimum for each individual run.

VERDICT INTERPRETATION
----------------------
Every simulated event was checked for election safety, log matching, leader
completeness, and applied-prefix agreement. No invariant violation occurred.

Every run invoked the Porcupine linearizability checker. Porcupine Illegal is
a violation and fails the run; there were zero Illegal verdicts. Porcupine
Unknown is an inconclusive bounded-search timeout and is non-failing in the
current harness. Batch output does not retain the Ok/Unknown split, so this
artifact does not claim that all 1,400 histories returned Ok.

The harness-caught applied-term issue was a checker-precision bug: election
safety used a stale last-applied-entry term as a proxy for current Raft term.
It was fixed to use raft.Node.Term(). It was not a Raft consensus bug.
